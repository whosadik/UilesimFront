## Цель шага

Интегрировать **ТОЛЬКО**:  CartPage.tsx и все его компоненты  


## Жёсткие правила

1. **Бэкенд НЕ менять** (никаких правок, миграций, новых эндпойнтов). Только читать код/схемы.
2. Убрать **моки/статические данные** и заменить на **реальные данные из API** — только где API уже это отдаёт.
3. Если UI просит поле/данные, которых **нет в API** — **не ломай UI**, оставь fallback/заглушку как сейчас и запиши это в “Backend notes”.
4. Делай **минимальные изменения**: без рефакторинга “на будущее”, только нужное для интеграции.
5. Соблюдай паттерны проекта: существующий api-client/wrappers, `ApiError`, обработка `401/403 -> /login`, `loading/error/retry`.
6. Если нужного frontend wrapper нет — можно добавить **тонкий wrapper** во фронтовом api-слое (без изменений бэка).
7. Перед правками проверь контракт по API-докам/схемам (источник истины: OpenAPI + schema map/index).   

## Процесс (обязателен)

### A) Анализ

* Выпиши, какие данные нужны UI (по props/рендеру).
* Найди подходящие эндпойнты и реальные поля ответа/запроса (по docs + при необходимости по backend serializers/views).
* Составь явный **маппинг UI-поля ↔ API-поля**.

### B) Реализация (только фронт)

* Удали мок/статические массивы/захардкоженные значения, которые заменяются API.
* Подключи существующие API wrappers (или добавь новый wrapper во фронте).
* Добавь состояния: `loading`, `error` (с retry), `401/403 -> /login`.
* Не меняй внешний вид UI и не трогай unrelated код.

### C) Самопроверка

* Убедись, что TypeScript не ругается.
* Прогони сборку/линт (минимум `npm run build` или эквивалент) и сообщи результат.

## Тексты/локализация (обязательное)

* НИКОГДА не генерируй “битые” заголовки/тексты (кракозябры, смешение кодировок, нечитаемые символы).
* Все новые строки UI (заголовки, кнопки, подсказки, ошибки) пиши **только на русском**, простым человеческим языком.
* Если на странице уже есть тексты — **не переписывай** их без необходимости. Меняй тексты только если это нужно для интеграции (например, сообщение об ошибке/загрузке).
* Любые fallback-тексты (когда данных нет) тоже должны быть на русском и понятны пользователю.
